# E2 硬件描述语言（一）

Verilog HDL 是一种硬件描述语言（HDL：Hardware Description Language），以文本形式来描述数字系统硬件的结构和行为的语言，用它可以表示逻辑电路图、逻辑表达式，还可以表示数字逻辑系统所完成的逻辑功能。

Verilog 可以从五个层次对电路（系统）进行描述，包括：系统级、算法级、寄存器传输级（即 RTL 级）、门级、开关级。我们平时用的最多的为 RTL 级，故 Verilog 代码也经常被称为 RTL 代码。

## Getting Started

Designing a circuit requires several steps:

- Writing HDL (Verilog) code
- compiling the code to produce a circuit
- then simulating the circuit and fixing bugs.

### Simulation

First, it reports whether your circuit matches the reference circuit exactly (zero "mismatches") or how many "mismatches" occurred.

Second, it may produce timing diagrams that show your circuit outputs when running our test vectors. The simulation waveform is grouped into three sections: "Inputs", "Yours", and "Ref". 

**The module name and port names of the top-level `top_module` must not be changed**

### Final Status

- **Compile Error** — Circuit did not compile.
- **Simulation Error** — Circuit compiled successfully, but simulation did not complete.
- **Incorrect** — Circuit compiled and simulated, but the outputs did not match the reference.
- **Success!** — Circuit was correct

Verilog 常量通常长这样：

```verilog
<位宽>'<进制><数值>
```

一个是在写“流程”，一个是在画“电路”。

## Verilog Language

### Basics

#### Wire

In a Verilog "continuous assignment" (`assign left_side = right_side;`), the value of the signal on the right side is driven onto the wire on the left side. （the assignment continues all the time even if the right side's value changes）

- `module` / `endmodule`

  表征模块的开始与结束。

- `example`

  模块名可由用户指定，可包含字母、数字及下划线，需以字母开头，区分大小写

- `assign`

  赋值操作关键字，该关键字后可跟一个赋值表达式，该关键字是实现组合逻辑操作的一种主要描述方式。

- `input` / `output`

  表征该信号的方向，除输入、输出外还有一种 `inout`（输入输出）型。

- 操作符

  `&`：按位与

  `|`：按位或

  `^`：按位异或

  `~`：按位取反

#### Wire4

When you have multiple `assign` statements, the **order** in which they appear in the code **does not matter**

- `[3:0]`

  表征该信号的位宽，实例中是推荐写法，`[0:3]`、`[4:1]` 等写法也是合法的

  ```verilog
  input [3:0] a
  ```

- `//`、`/* */`

  代码注释，增加代码可读性，`//` 为单行注释，`/* */` 为多行注释

#### Inverter / Notgate

Use an assign statement. The `assign` statement will *continuously drive* the inverse of `in` onto wire `out`.

`~`

#### NOR gate

或非门

```verilog
assign out=~(a|b);
```

#### XNOR gate

同或门

```verilog
assign out = ~(a ^ b);
```

#### Declaring wires

As circuits become more complex, you will need wires to connect internal components together. When you need to use a wire, you should declare it in the body of the module, somewhere before it is first used.

**声明和赋值最好分开写**

```verilog
module top_module (
    input in,              // Declare an input wire named "in"
    output out             // Declare an output wire named "out"
);

    wire not_in;           // Declare a wire named "not_in"

    assign out = ~not_in;  // Assign a value to out (create a NOT gate).
    assign not_in = ~in;   // Assign a value to not_in (create another NOT gate).

endmodule   // End of module "top_module"
```

### Vectors

多根打包在一起的线

**表示多位数据**

比如数字、地址、指令、寄存器内容、总线数据

#### Vector0

Vectors are used to group related signals using one name to make it more convenient to manipulate. For example, `wire [7:0] w;` declares an 8-bit vector named `w` that is functionally equivalent to having 8 separate wires.

| 语言    | 写法                    |
| ------- | ----------------------- |
| C       | `int arr[100]`          |
| Verilog | `wire [99:0] my_vector` |

**声明 vector 时维度在前面，但访问元素时下标在后面**

#### Vector1

```verilog
type [upper:lower] vector_name;

input wire [3:-2] z;  // 6-bit wire input (negative ranges are allowed)
output [3:0] a;       // 4-bit output wire. Type is 'wire' unless specified otherwise.
wire [0:7] b;         // 8-bit wire where b[0] is the most-significant bit.
```

工程中基本都会统一写成：

```verilog
[MSB:LSB]
```

**vector 一旦声明了方向，就必须始终按同一个方向使用。**

##### Indexed Vector Part Select

```verilog
[start +: width]//从 start 开始，往高位取 width 位
[start -: width]//从 start 开始，往低位取 width 位
```

##### Packed array

```
reg [7:0] data
```

表示 **8 bit vector**。

------

##### Unpacked array

```
reg [7:0] mem [255:0]
```

表示 **256 × 8bit memory**。数组，256个元素，每个元素8bit

#### Bitwise operators

A bitwise operation between two N-bit vectors replicates the operation for each bit of the vector and produces a N-bit output, while a logical operation treats the entire vector as a boolean value (true = non-zero, false = zero) and produces a 1-bit output.

**Verilog 不允许在语句中间声明变量。**

#### Vector concatenation operator

The concatenation operator `{a,b,c}` is used to create larger vectors by concatenating smaller portions of a vector together.

```verilog
{3'b111, 3'b000} => 6'b111000
{1'b1, 1'b0, 3'b101} => 5'b10101
```

The concatenation operator can be used on both the left and right sides of assignments.

```verilog
input [15:0] in;
output [23:0] out;
assign {out[7:0], out[15:8]} = in;         // Swap two bytes. Right side and left side are both 16-bit vectors.
assign out[15:0] = {in[7:0], in[15:8]};    // This is the same thing.
assign out = {in[7:0], in[15:8]};       // This is different. The 16-bit vector on the right is extended to
                                        // match the 24-bit vector on the left, so out[23:16] are zero.
                                        // In the first two examples, out[23:16] are not assigned.
```

#### Replication operator

The replication operator allows repeating a vector and concatenating them together:

```verilog
{num{vector}}
```

```verilog
{5{1'b1}}           // 5'b11111 (or 5'd31 or 5'h1f)
```

### Modules: Hierarchy

#### Module

 a `module`, is a circuit that interacts with its outside through input and output ports

Verilog 有两种端口连接方式：

1️⃣ **按位置连接（By position）**
 2️⃣ **按名称连接（By name）**

------

##### 按位置连接（By Position）

语法：

```
mod_a instance1 ( wa, wb, wc );
```

如果模块定义是：

```verilog
module mod_a (
    input in1,
    input in2,
    output out
);
```

连接规则：

```
按端口声明顺序，从左到右
```

图示：

```
wa ----> in1
wb ----> in2
wc <---- out
```

------

##### 按名称连接（By Name）

语法：

```verilog
mod_a instance2 (
    .in1(wa),
    .in2(wb),
    .out(wc)
);
```

语法结构：

```verilog
.端口名(信号名)
```

解释：

```text
mod_a port      top module signal
--------------------------------
in1             wa
in2             wb
out             wc
```

### Procedures

#### Always blocks (combinational)

`always`语句用于描述硬件电路中持续或重复执行的行为，是定义时序逻辑和组合逻辑的核心结构。

- Combinational: `always @(*)`
- Clocked: `always @(posedge clk)`

组合逻辑的 `always` 块，本质上和 `assign` 是等价的。

```verilog
assign out1 = a & b | c ^ d;
```

和

```verilog
always @(*) 
    out2 = a & b | c ^ d;
```


描述的是 同一个组合逻辑电路。

**主要看哪种写法更方便。**

一般经验是：

简单逻辑 → `assign`   复杂逻辑（if...else...) → `always`

在 `always` 里你可以写：

```verilog
if
case
for
begin/end
```

**组合逻辑一定要写**(*)

```verilog
always @(*)
```

#####  wire与reg

- assign 左边必须是 wire

- always 左边必须是 reg

- **wire / reg 不代表真实硬件，只是语法规则**

#### Always blocks (clocked)

| 类型         | 写法           | 使用位置  | 典型用途     |
| ------------ | -------------- | --------- | ------------ |
| 连续赋值     | `assign x = y` | always 外 | 简单组合逻辑 |
| blocking     | `x = y`        | always 内 | 组合逻辑     |
| non-blocking | `x <= y`       | always 内 | 时序逻辑     |

```veri
assign → wire
always @(*) → reg + =
always @(posedge clk) → reg + <=
```

用<= 像并行执行的电路。右边表达式会先被同时计算，但赋值操作会在整个块执行完毕后统一进行。

- `posedge`

  Verilog 关键字，表示上升沿的意思。`always @(posedge clk)` 表示在 clk 信号的上升沿的时刻，执行 always 块内部的语句，与此相对应的，是表示下降沿的关键字 negedge。凡是带有 posedge 或 negedge 的 always 块，都会被综合成时序逻辑电路。

#### If statement

An `if` statement usually creates a 2-to-1 multiplexer, selecting one input if the condition is true, and the other input if the condition is false.

```verilog
always @(*) begin
    if (condition) begin
        out = x;
    end
    else begin
        out = y;
    end
end
```

```verilog
assign out = (condition) ? x : y;
```

组合逻辑里如果某些分支不给输出赋值，输出就会“记住上一次”，这就是锁存器现象

正确做法是：**组合逻辑里把所有情况都覆盖掉。**

#### Case statement

```verilog
always @(*) begin     // This is a combinational circuit
    case (in)
      1'b1: begin 
               out = 1'b1;  // begin-end if >1 statement
            end
      1'b0: out = 1'b0;
      default: out = 1'bx;
    endcase
end
```

- The case statement begins with `case` and each "case item" ends with a colon. There is no "switch".
- Each case item can execute *exactly one* statement. This makes the "break" used in C unnecessary. But this means that if you need more than one statement, you must use `begin ... end`.
- **不要忘记`endcase`**

#### Priority encoder with casez

This is what `casez` is for: It treats bits that have the value `z` as don't-care in the comparison.

```verilog
always @(*) begin
    casez (in[3:0])
        4'bzzz1: out = 0;   // in[3:1] can be anything
        4'bzz1z: out = 1;
        4'bz1zz: out = 2;
        4'b1zzz: out = 3;
        default: out = 0;
    endcase
end
```

- There is also a similar `casex` that treats both `x` and `z` as don't-care. I don't see much purpose to using it over `casez`.
- The digit `?` is a synonym for `z`. so `2'bz0` is the same as `2'b?0`

To avoid creating latches, **all outputs must be assigned a value in all possible conditions** . Simply having a `default` case is not enough. You must assign a value to all four outputs in all four cases and the default case.

```verilog
always @(*) begin
    up = 1'b0; down = 1'b0; left = 1'b0; right = 1'b0;
    case (scancode)
        ... // Set to 1 as necessary.
    endcase
end
```

### More Verilog Features

#### conditional ternary operator

```verilog
(condition ? if_true : if_false)
```

```verilog
always @(posedge clk)         // A T-flip-flop.
  q <= toggle ? ~q : q;
```

#### Reduction operators

对一个向量里的所有位做整体运算

```verilog
& a[3:0]     // AND: a[3]&a[2]&a[1]&a[0]. Equivalent to (a[3:0] == 4'hf)
| b[3:0]     // OR:  b[3]|b[2]|b[1]|b[0]. Equivalent to (b[3:0] != 4'h0)
^ c[2:0]     // XOR: c[2]^c[1]^c[0]
```

偶奇偶校验

```verilog
module top_module (
    input  [7:0] in,
    output parity
);

assign parity = ^in;

endmodule
```

XOR 的性质是：

- 奇数个 `1` → 结果 `1`
- 偶数个 `1` → 结果 `0`

#### 批量实例化模块`generate`

`generate` 表示：
 这一段不是普通运行时语句，而是 **在编译 / 综合时展开硬件**。

比如你写：

```verilog
generate
    for(i=1; i<100; i=i+1) begin : name1
        full_adder adder_i(...);
    end
endgenerate
```

综合器不会“循环执行 99 次程序”，而是直接把它展开成类似这样：

```verilog
full_adder adder_1(...);
full_adder adder_2(...);
full_adder adder_3(...);
...
full_adder adder_99(...);
```

也就是说，`generate` 本质上是：

- 帮你少写重复代码
- 最终还是生成一堆真实的硬件实例

---

`genvar` 就是 **generate-for 循环专用的循环变量**。

例如：

```verilog
genvar i;
```

---

`name1` 的作用你可以理解成：

> 给 for 生成出来的每一层外壳起个名字，避免混乱。

有点像这样：

```
name1[1] 里面放一个 adder_i
name1[2] 里面放一个 adder_i
...
```

所以虽然内部都叫 `adder_i`，但外面壳子不同，就不冲突。

##### 模板

```verilog
genvar i;
generate
    for(i=0; i<N; i=i+1) begin : block_name
        some_module u_module (
            .in (...),
            .out(...)
        );
    end
endgenerate
```
