# Installing Tools

download and install a software by one command.This is achieved by the package manager. In Ubuntu, the package manager is called `apt`.

### Setting APT source file

Run the following commands to update the APT source file:

```sh
sed -i "s/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g" /etc/apt/sources.list
```

One solution is first switching to the root account. But to avoid switching, an alternative way is to use `sudo`. If you find an operation requires superuser permission, append `sudo` before that operation.

在 Ubuntu（以及很多现代 Linux 发行版）中，**root 账户默认是没有密码且被锁定的**。

**执行修改密码指令：**

```
sudo passwd root
```

*(这一步的意思是：以普通用户的特权身份，去修改 root 的密码)*

**输入【你的用户密码】：**
 系统会先问 `[sudo] password for [你的用户名]:`。这时输入你登录系统时的那个密码。
 *(注意：依然是“隐形墨水”，屏幕没显示，输完直接回车)*

**设置【新的 Root 密码】：**
 如果成功，系统会提示 `Enter new UNIX password:`。

### 🧠 核心心法：三大模式切换

在 Vim 中，永远要清楚自己当前处于什么模式。遇到任何诡异的情况，**狂按几下 `Esc`** 就能保平安。

| 目标模式                 | 触发键               | 就像是...            | 作用说明                                                     |
| ------------------------ | -------------------- | -------------------- | ------------------------------------------------------------ |
| 🔙 **普通模式** (Normal)  | `Esc`                | 放下笔，准备发号施令 | 万物归宗的默认状态。此时敲击键盘执行的是“移动、复制、删除”等指令。 |
| ✍️ **插入模式** (Insert)  | `i` 或 `a` 或 `o`    | 拿起笔，开始写字     | `i` (Insert) 光标前插入； `a` (Append) 光标后插入； `o` (Open) 在下方新开一行并进入编辑。 |
| ⚙️ **命令模式** (Command) | `:` (普通模式下输入) | 拿起对讲机呼叫总部   | 激活底部的命令行，用于执行“保存、退出、替换”等全局操作。     |

### 🕹️ 极客步伐：光标移动 (普通模式下使用)

忘掉你的鼠标和键盘上的上下左右方向键吧！习惯用字母键移动，是让你的手速起飞的关键。

| 动作                | 指令            | 记忆口诀与实战意义                                           |
| ------------------- | --------------- | ------------------------------------------------------------ |
| **左 下 上 右**     | `h` `j` `k` `l` | 手指完全不需要离开主键盘区。`j` 长得最像向下的箭头，`k` 向上。 |
| **跳至行首 / 行尾** | `^` / `$`       | 程序员的直觉：`^` 是开头，`$` 是结尾。修改超长代码行时极度好用。 |
| **飞到文件头**      | `gg`            | Go Go！直接跳到文件的第一行。                                |
| **沉到文件底**      | `G`             | 大写的 G，一沉到底。看报错日志时经常需要跳到最后一行。       |
| **跳到指定行**      | `行号 + G`      | 比如编译器报错“第42行有语法错误”，直接输入 `42G` 就能空降过去。 |
| **显示行号**        | `Ctrl+G`        |                                                              |
| **选择部分**        | `v`             | 按下v，配合光标移动，可实现选择                              |

### 🔪 文本手术刀：复制、粘贴与删除 (普通模式下使用)

在 Vim 里，“删除”本质上就是“剪切”。这把手术刀非常锋利，配合撤销键使用风味更佳。

| 动作                   | 指令       | 记忆口诀与实战意义                                           |
| ---------------------- | ---------- | ------------------------------------------------------------ |
| **复制**               | `y`        |                                                              |
| **复制当前行**         | `yy`       | Yank (拽走) 一行代码。如果输入 `3yy` 就是往下复制 3 行。     |
| **粘贴**               | `p`        | Paste (粘贴) 到光标所在行的下一行。                          |
| **删除 (剪切) 当前行** | `dd`       | Delete 一行。删完之后按 `p`，你会发现刚刚删掉的内容被粘贴出来了。输入 `5dd` 就是连删 5 行。 |
| **删除一个单词**       | `dw`       | Delete Word。光标放在单词开头，瞬间删掉这个词。              |
| 💊 **撤销 (后悔药)**    | `u`        | Undo。**Vim 里最伟大的按键！** 只要按错了、删错了，立刻按 `u` 就能时光倒流。 |
| 🔄 **重做 (取消撤销)**  | `Ctrl + r` | Redo。如果 `u` 按得太多，把正确的也撤销了，用它恢复。        |

####  管理你的“陷阱”：查看与删除 

断点设多了可能会乱，学会管理它们非常重要。

| 目标             | 指令               | 就像是...                                       |
| ---------------- | ------------------ | ----------------------------------------------- |
| **查看所有断点** | **`info b`**       | 调出地图，看看我在哪儿埋了雷。                  |
| **删除特定断点** | **`d 断点编号`**   | 拆除编号为 N 的雷（编号可通过 `info b` 查到）。 |
| **删除所有断点** | **`d`**            | 瞬间清空战场，一个雷都不留。                    |
| **临时禁用断点** | **`dis 断点编号`** | 关掉引信，雷还在，但暂时不会炸。                |

### Installing tmux

在 tmux 里，所有的快捷指令都需要一个**“前置唤醒键”（Prefix）**，默认是 **`Ctrl + b`**。你必须先按下它，松开，然后再按别的键。

| 你的目标         | 具体操作 (严格按顺序)                             | 视觉效果                           |
| ---------------- | ------------------------------------------------- | ---------------------------------- |
| **左右分屏**     | 先按 `Ctrl + b`，松开，再按 `%`                   | 屏幕从中间裂开，左右各一个终端。   |
| **上下分屏**     | 先按 `Ctrl + b`，松开，再按 `"`                   | 屏幕上下裂开。                     |
| **在分屏间跳跃** | 先按 `Ctrl + b`，松开，再按 **方向键 (上下左右)** | 光标在不同的窗口间穿梭。           |
| **关闭当前分屏** | 在当前分屏输入 `exit` 并回车，或者按 `Ctrl + d`   | 这个小窗口被销毁，空间还给其他窗口 |

# Getting Source Code for PAs

通过`git log tracer-ysyx`查看代码跟踪日志

When you want to commit the change, type

```shell
git add .
git commit --allow-empty
```

### 1. 💾 你的日常“存盘”三连击

这三个指令是你以后每天都要敲无数遍的组合拳。当你完成了一个小功能（比如写好了一个乘法指令），请务必存盘！

| Git 指令                           | 动作含义                               | 就像是...                                                    |
| ---------------------------------- | -------------------------------------- | ------------------------------------------------------------ |
| **`git status`**                   | 查看当前仓库的状态                     | 打开游戏状态栏，看看：“我刚才改了哪些文件？有没有未保存的内容？” |
| **`git add .`**                    | 把当前目录下所有改动的文件放入“暂存区” | 把你刚才打怪掉落的战利品，统统装进背包准备带走。             |
| **`git commit -m "实现乘法指令"`** | 正式生成一条存档记录（附带注释）       | 找旅店老板正式存档，并给这个存档取个名字。                   |

> **💡 PA 实验小贴士：** 在“一生一芯”中，实验框架会经常通过 `make` 命令在后台**自动帮你 commit** 以记录你的进度。但手动存盘的好习惯依然至关重要！

------

### 2. ⏳ 时光倒流与侦探行动

当你写了一下午代码，突然发现编译报错，并且完全不知道自己改坏了哪里时，这几招能救你的命。

| Git 指令                 | 动作含义                                       | 就像是...                                                    |
| ------------------------ | ---------------------------------------------- | ------------------------------------------------------------ |
| **`git log`**            | 查看历史存档记录                               | 翻开冒险日志，看看你过去每天都干了什么（按 `q` 退出查看）。  |
| **`git diff`**           | 查看当前没存盘的代码和上次存盘的代码有什么区别 | 侦探用的放大镜，高亮显示你刚才到底删了哪行、加了哪行代码（红绿底色显示）。 |
| **`git restore 文件名`** | 撤销对某个文件的修改，恢复到上次存盘的样子     | 这瓶药水配错了，直接把瓶子砸了，重新拿一瓶原始配方。         |

------

### 3. 🌌 分支管理

“一生一芯”的 PA 实验是一环扣一环的（PA0 -> PA1 -> PA2...）。为了保证你在做新实验时，不小心把旧实验跑通的代码搞崩，你需要学会使用**分支（Branch）**。

| Git 指令                  | 动作含义                                | 就像是...                                                    |
| ------------------------- | --------------------------------------- | ------------------------------------------------------------ |
| **`git branch`**          | 查看当前所有的分支                      | 看看你当前打出了几个平行结局。带 `*` 号的是你当前所处的宇宙。 |
| **`git checkout -b pa1`** | **创建并切换**到一个名为 `pa1` 的新分支 | 复制当前的世界线，在新的时间线上开始你的 PA1 实验冒险，无论怎么折腾都不会影响 PA0。 |
| **`git checkout master`** | **切换**回名为 `master` 的主分支        | 从平行宇宙穿越回主线剧情。                                   |

------

### 4. ☁️ 云端同步与防丢策略

你花了几周写的心血，绝对不能只存在本地虚拟机里（万一虚拟机崩了就全完了）。你需要经常把代码同步到你之前配置好 SSH 密钥的 GitHub 上。

| Git 指令       | 动作含义                         | 就像是...                                            |
| -------------- | -------------------------------- | ---------------------------------------------------- |
| **`git push`** | 把本地的存档推送到云端（GitHub） | 将你的游戏存档上传到云端服务器，这下电脑炸了也不怕。 |
| **`git pull`** | 从云端把最新的代码拉取到本地     | 换了台电脑，把云端存档下载下来继续玩。               |
