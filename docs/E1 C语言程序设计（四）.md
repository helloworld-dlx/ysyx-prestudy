#### 24.1.2 `malloc`与`free`

C标准库函数`malloc`可以在堆空间动态分配内存，动态分配的内存用完之后可以用`free`释放

```c
#include <stdlib.h>

void *malloc(size_t size);
返回值：成功返回所分配内存空间的首地址，出错返回NULL

void free(void *ptr);
```

`malloc`的参数`size`表示要分配的字节数，如果分配失败（可能是由于系统内存耗尽）则返回`NULL`

---

想象一下，内存是一间巨大的自助餐厅。

- **栈（Stack）：** 是你座位上那套餐具，大小固定，吃完饭服务员会自动收走。
- **堆（Heap）：** 是餐厅中央那个巨大、壮观的**餐盘供应区**，有各种大小的盘子。你可以随时去拿，但**你必须自己负责归还**。

| 概念         | 核心逻辑                                 | 餐厅比喻                                     | 必须遵守的规则                                               |
| ------------ | ---------------------------------------- | -------------------------------------------- | ------------------------------------------------------------ |
| **`malloc`** | 在**堆**上动态申请一块指定大小的内存。   | 去餐盘区，按需领一个盘子。                   | 1. 检查返回值是否为 `NULL`。 2. 不能假设内存是干净的。       |
| **`free`**   | 释放之前 `malloc` 申请的内存，还给系统。 | 把用完的盘子还给回收处。                     | 1. `free` 之后，必须把原指针设为 `NULL`。                    |
| **内存泄漏** | 申请了内存，但用完后**忘记 `free`**。    | 拿了盘子没还，直接带出餐厅。                 | **申请和释放必须成对出现！** `malloc` 多少次，就要 `free` 多少次。 |
| **悬垂指针** | 指向的内存已经被 `free` 掉了的指针。     | 还了盘子，但手里还攥着那张写着地址的旧纸条。 | `free` 之后，必须 `p = NULL;`。                              |

### 24.2 传入参数与传出参数

如果函数接口有指针参数，既可以把指针所指向的数据传给函数使用（称为传入参数），也可以由函数填充指针所指的内存空间，传回给调用者使用（称为传出参数）

| 概念         | 核心目的             | 动作                | 通俗比喻                 |
| ------------ | -------------------- | ------------------- | ------------------------ |
| **传入参数** | 把数据**给**函数用   | **读** (Read-Only)  | 把**填好的**表格给别人看 |
| **传出参数** | 从函数**拿**结果回来 | **写** (Write-Only) | 把**空白的**表格给别人填 |

**传入参数是函数括号里的“原材料”；而传出结果有两种方式，简单的用 `return`，复杂和多个的就得靠指针类型的“传出参数”来搞定！**
