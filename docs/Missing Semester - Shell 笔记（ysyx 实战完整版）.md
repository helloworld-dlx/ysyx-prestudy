# Missing Semester - Shell 笔记（ysyx 实战完整版）

------

## 一、Shell 的本质

Shell 是一个**命令解释器**，作用是：

- 读取你输入的命令
- 找到对应程序并解析带上的属性（参数）
- 执行它并返回结果

------

## 二、路径与目录（基础）

### 核心概念

- `.` 当前目录
- `..` 上一级目录
- `~` 家目录 (通常是 `/home/用户名`)
- `/` 根目录

### 常用命令及核心属性

- **`pwd`**：显示当前绝对路径 (Print Working Directory)
- **`cd`**：切换目录
  - `cd -`：回到上一次所在的目录（在两个目录间反复横跳时极好用）
- **`ls`**：列出文件
  - **`-l`**：长格式显示（看权限、拥有者、文件大小、修改时间）
  - **`-a`**：显示所有文件，包括隐藏文件（如 PA 里的 `.git`、`.vimrc`）
  - **`-h`**：人类可读格式（把 1048576 变成 1M，常配合 `-l` 使用：`ls -lh`）
  - **`-t`**：按修改时间排序（常配合 `-l`：`ls -lt`，把刚改过的文件排前面）

------

## 三、文件操作

- **`cp`**：复制 (Copy)
  - **`-r`**：递归复制（连同文件夹里的所有子文件夹和文件一起复制）
  - **`-i`**：覆盖前提示确认（防手抖）
- **`mv`**：移动/重命名 (Move)
  - 用法：`mv 原文件名 新文件名` (重命名) / `mv 文件名 目录/` (移动)
- **`rm`**：删除 (Remove)
  - **`-r`**：递归删除文件夹
  - **`-f`**：强制删除，不提示（⚠️ PA 实验高危操作，删代码前三思）
  - 组合：`rm -rf dir/` (极其凶残的强制删库命令)
- **`mkdir`**：创建目录
  - **`-p`**：递归创建多级目录（例如 `mkdir -p a/b/c`，即使 a 和 b 不存在也会一并建好）

------

## 四、查看文件内容

- **`cat`**：全部输出到屏幕
  - **`-n`**：显示行号（代码报错说在第几行时，用来核对极佳）
- **`less`**：分页查看（看长篇 log 必备，比 `more` 更好用）
  - `空格` / `f`：向下翻页
  - `b`：向上翻页
  - `/关键字`：向下搜索关键字（按 `n` 找下一个，`N` 找上一个）
  - `q`：退出
- **`head` / `tail`**：看头看尾
  - **`-n 行数`**：显示前/后多少行（如 `tail -n 20 log.txt`）
  - **`-f`**：（tail 专属神技）实时滚动监控文件末尾（比如另开一个终端看编译日志）

------

## 五、查找关键字（grep - 核心中的核心）

在浩如烟海的 NEMU 源码里找某个寄存器或指令定义，全靠它。

- **`grep "关键字" 文件名`**
  - **`-n`**：显示匹配所在的行号（必须加，不然找不到在哪改代码）
  - **`-r`**：递归搜索当前目录下的所有文件（`grep -r "main" .`）
  - **`-i`**：忽略大小写
  - **`-v`**：反向匹配（找出**不包含**该关键字的行，常用于过滤掉乱七八糟的 warning）
  - **`-w`**：精确匹配整个单词（搜 `pc` 时，不会把 `space` 也搜出来）

------

## 六、重定向与管道（重点）

### 1. 重定向（控制数据流向）

- **`>`**：覆盖写入（标准输出）
- **`>>`**：追加写入（不覆盖原文件）
- **`2>`**：仅重定向错误输出（比如只把编译报错存起来）
- **`2>&1`**：将错误输出（2）重定向到标准输出（1），通常结合 `>` 一起用。
  - 极其常用：`make > build.log 2>&1`（把所有正确和错误信息都打包装进 log）

### 2. 管道 `|`

把左边命令的输出，直接作为右边命令的输入。

- 组合：`make 2>&1 | grep "error"`（编译时屏幕只显示包含 error 的行）

------

## 七、tee（边看边存）

既要把输出打印在屏幕上实时看，又要存进文件里留底。

- 用法：`cmd | tee log.txt`
- 追加用法：`cmd | tee -a log.txt` (相当于 `>>`)

------

## 八、权限与执行

Linux 下文件不是有后缀名（如 .exe）就能运行的，必须有“执行权限”。

- **`chmod`**：修改权限
  - **`+x`**：赋予执行权限（`chmod +x script.sh`）
  - **`-R`**：递归修改整个目录的权限
- **shebang（关键）**：
  - 脚本第一行必须写：`#!/bin/bash`
  - 作用：告诉系统用什么解释器来运行这个文件。

------

## 九、Shell 参数（脚本）

写自动化测试脚本时必须认识的符号：

- `$0`：脚本自己的名字
- `$1`, `$2`...：传入的第 1、2 个参数（如 `./run.sh nemu`，`$1` 就是 nemu）
- `$@`：传入的所有参数
- `$#`：传入的参数总个数
- **`$?`**：**极其重要！** 上一条命令的退出状态码（0 代表成功，非 0 代表出错。NEMU 跑没跑通可以看它）。

------

## 十、命令执行控制

把多个命令连在一起，实现自动化。

- **`&&`**：逻辑与。前面成功了，才执行后面。（`make && ./nemu`：编译成功才运行）
- **`||`**：逻辑或。前面失败了，才执行后面。（`make || echo "编译炸了！"`）
- **`;`**：无脑按顺序执行。不管前面成败，后面必须执行。（`make clean ; make`）

------

## 十一、环境变量

系统找命令的路径字典。

- **`echo $PATH`**：看看系统目前会在哪些文件夹里找命令
- **`which gcc`**：告诉你目前调用的 gcc 到底在哪个绝对路径下
- **`export`**：设置环境变量
  - 用法：`export PATH=$PATH:/你的/新路径`（PA 环境配置中经常要把 riscv 的工具链加进去）

------

## 十二、查找文件（find）

当你不记得自己把 `.c` 文件建在哪了：

- **`find 目录 -name "名字"`**：按名字找（`find . -name "*.c"` 找所有 C 文件）
- **`-type`**：按类型找（`-type f` 找文件，`-type d` 找目录）
- **`-mtime`**：按修改时间找（`-mtime -1` 找最近 1 天改过的文件）

------

## 十三、最常用组合（ysyx 实战级）

### 1️⃣ 自动化编译 + 强迫症清理

```
make clean && make -j4 2>&1 | tee build.log
```

*(注：`-j4` 代表开 4 个线程加速编译)*

### 2️⃣ 绝望找 Bug (只看错误)

```
make 2>&1 | grep -i "error"
```

### 3️⃣ 运行 NEMU 并留存犯罪现场

```
./nemu 2>&1 | tee run.log
```

### 4️⃣ 全局追踪一个变量在哪被修改过

```
grep -rnw "paddr_read" .
```

## 十四、最小脚本模板（直接用）

### PA 自动化跑批测试模板 (`test.sh`)

```
#!/bin/bash
# 自动清理并编译，如果失败直接退出
make clean ; make -j4 || exit 1 

# 运行程序并接收外部参数，记录日志
echo "开始运行..."
./nemu "$1" 2>&1 | tee run.log

# 检查上一条命令(nemu)是否正常退出
if [ $? -eq 0 ]; then
    echo "运行成功(HIT GOOD TRAP)!"
else
    echo "运行失败(HIT BAD TRAP或崩溃)!"
fi
```

## 十五、常见问题定位（救火指南）

### 1. 命令找不到 (Command not found)

- **排查**：`which xxx` 看有没有。没有就 `apt-get install`，有但提示找不到就查 `echo $PATH`。

### 2. 脚本拒绝执行 (Permission denied)

- **排查**：`ls -l script.sh` 看看有没有 `x` 权限。
- **解决**：`chmod +x script.sh`

### 3. Make 报错但信息滚得太快看不清

- **解决**：`make 2>&1 | less` （用上下键慢慢找第一个 Error 出现在哪）

## 十六、核心工作流（一生一芯日常）

**`cd ysyx-workbench/nemu`** (进入战场)
 → **`vim src/...`** (修代码)
 → **`make`** (编译)
 → **`make run 2>&1 | tee log.txt`** (运行存底)
 → **`grep "panic" log.txt`** (找崩溃点)
 → **`git add . && git commit -m "修复xx"`** (存档保平安)
 → **重复上述步骤直到头秃/通关**。
