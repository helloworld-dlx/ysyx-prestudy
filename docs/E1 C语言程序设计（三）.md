## 第21章 预处理

### 21.2 宏定义

变量式宏定义：`#define N 20`或`#define STR "hello, world"`这种宏定义

#### 21.2.1 函数式宏定义

像函数调用一样在代码中使用，称为函数式宏定义（Function-like Macro）

```c
#define MAX(a, b) ((a)>(b)?(a):(b))
k = MAX(i&0x0f, j&0x0f)
```

看第二行的表达式展开成什么样，可以用`gcc`的`-E`选项或`cpp`命令

**经验法则：**
 **在函数式宏的参数中，绝对不要使用带有副作用（Side Effect）或计算成本高昂的表达式，比如 `i++`、`--j` 或者递归函数调用。**

尽管函数式宏定义和真正的函数相比有很多缺点，但只要小心使用还是会显著提高代码的执行效率，毕竟省去了分配和释放栈帧、传参、传返回值等一系列工作，因此那些简短并且被频繁调用的函数经常用函数式宏定义来代替实现。

| 宏的封装方式          | 优点                 | 缺点                                    | 结论                                                         |
| --------------------- | -------------------- | --------------------------------------- | ------------------------------------------------------------ |
| **不封装**            | 简单                 | 在 `if` 中只控制第一句，极易出错        | 绝对禁止                                                     |
| **`{ ... }`**         | 解决了 `if` 的问题   | 破坏了 `if-else` 结构（因为多余的分号） | 严禁使用                                                     |
| **`do{...}while(0)`** | 语法上是**单条语句** | 写起来有点怪                            | **完美！** 它能“吃掉”调用宏时最后那个分号，使宏在任何地方都表现得像一个真正的函数。 |

---

如果在一个程序文件中重复定义一个宏，C语言规定这些重复的宏定义必须一模一样

如果需要重新定义一个宏，和原来的定义不同，可以先用`#undef`取消原来的定义，再重新定义，例如：

```c
#define X 3
... /* X is 3 */
#undef X
... /* X has no definition */
#define X 2
... /* X is 2 */
```



#### 21.2.2 内联函数

新关键字`inline`，用于定义内联函数（inline function）

`inline`关键字告诉编译器，这个函数的调用要尽可能快，可以当普通的函数调用实现，也可以用宏展开的办法实现

```c
inline int MAX(int a, int b)
{
	return a > b ? a : b;
}
```

#### 21.2.3 `#`、`##`运算符和可变参数

在函数式宏定义中，`#`运算符用于创建字符串，`#`运算符后面应该跟一个形参（中间可以有空格或Tab），例如：

```c
#define STR(s) # s
STR(hello 	world)
```

用`cpp`命令预处理之后是`"hello␣world"`，自动用`"`号把实参括起来成为一个字符串，并且实参中的连续多个空白字符被替换成一个空格。

---

在宏定义中可以用`##`运算符把前后两个预处理Token连接成一个预处理Token，和`#`运算符不同，`##`运算符不仅限于函数式宏定义，变量式宏定义也可以用。例如：

```c
#define CONCAT(a, b) a##b
CONCAT(con, cat)
```

预处理之后是`concat`。再比如，要定义一个宏展开成两个`#`号，可以这样定义：

```c
#define HASH_HASH # ## #
```

---

1. 函数式宏定义也可以带可变参数，同样是在参数列表中用`...`表示可变参数。可变参数的部分用`__VA_ARGS__`表示，实参中对应`...`的几个参数可以看成一个参数替换到宏定义中`__VA_ARGS__`所在的地方

   ```c
   #define showlist(...) printf(#__VA_ARGS__)
   #define report(test, ...) ((test)?printf(#test):\
   	printf(__VA_ARGS__))
   showlist(The first, second, and third items.);
   report(x>y, "x is %d but y is %d", x, y);
   ```

   预处理之后变成：

   ```c
   printf("The first, second, and third items.");
   ((x>y)?printf("x>y"): printf("x is %d but y is %d", x, y));
   ```

2. | 宏定义              | 调用示例  | 预处理器视角                 | 结论                       |
   | ------------------- | --------- | ---------------------------- | -------------------------- |
   | `#define FOO()`     | `FOO()`   | 0 个参数，必须匹配 0 个      | 数量必须严格一致           |
   | `#define FOO(a)`    | `FOO()`   | 1 个参数，收到了一个“空参数” | **空参数**是合法的         |
   | `#define FOO(a,b)`  | `FOO(1,)` | 2 个参数，第二个是“空参数”   | 逗号是**占位符**，不能省略 |
   | `#define FOO(a,b)`  | `FOO(1)`  | **错误！** 参数数量不匹配    | 逗号数量必须匹配           |
   | `#define F(a, ...)` | `F(1)`    | 可变部分 `__VA_ARGS__` 为空  | 可变参数部分可以为空       |

### 21.3 条件预处理指示

| 指令                   | 含义                       | 核心用途                                | 比喻                               |
| ---------------------- | -------------------------- | --------------------------------------- | ---------------------------------- |
| **`#ifdef NAME`**      | 如果 `NAME` 被定义过       | **功能开关**，选择性地编译某段代码      | 检查配置单上**有没有**这个贴纸     |
| **`#ifndef NAME`**     | 如果 `NAME` **没**被定义过 | **头文件保护**，防止重复包含            | 检查配置单上**是不是没有**这个贴纸 |

| **`#elif expression`** | 否则如果...                | `else if` 的预处理版本                  | `else if`                          |
| **`#endif`**           | 结束条件块                 | 标记条件预处理的结束                    | 结束                               |

## 第23章 指针

### 23.1 指针的基本概念

把一个变量所在的内存单元的地址保存在另外一个内存单元中，保存地址的这个内存单元称为**指针**

`int *pi = &i;`表示定义一个指向`int`型的指针变量`pi`。

`&`是取地址运算符（Address Operator），`*`号是指针间接寻址运算符（Indirection Operator），`*pi`表示取指针`pi`所指向的变量的值

在同一个语句中定义多个指针变量，每一个都要有`*`号

如果要让`pi`指向另一个整型变量`j`，可以重新对`pi`赋值：

```c
pi = &j;
```

如果要改变`pi`所指向的整型变量的值，比如把变量`j`的值增加10，可以写：

```c
*pi = *pi + 10;
```

指针之间可以相互赋值，也可以用一个指针初始化另一个指针（**注意：两个指针必须是同一类型的**），例如：

```c
int *ptri = pi;
```

指向不确定地址的指针称为**“野指针”（Unbound Pointer）**，为避免出现野指针，在定义指针变量时就应该给它明确的初值，或者把它初始化为`NULL`（空指针）

`void *`指针与其它类型的指针之间可以隐式转换，而不必用类型转换运算符

### 23.2 指针类型的参数和返回值

```c
#include <stdio.h>

int *swap(int *px, int *py)
{
	int temp;
	temp = *px;
	*px = *py;
	*py = temp;
	return px;
}

int main(void)
{
	int i = 10, j = 20;
	int *p = swap(&i, &j);
	printf("now i=%d j=%d *p=%d\n", i, j, *p);
	return 0;
}
```

调用函数的传参过程相当于用实参定义并初始化形参，`swap(&i, &j)`这个调用相当于：

```c
int *px = &i;
int *py = &j;
```

| 方式         | 传递的是什么   | 函数内的操作       | 对原变量的影响 |
| ------------ | -------------- | ------------------ | -------------- |
| **直接传参** | 变量的**数值** | 改变副本           | **无效**       |
| **传指针**   | 变量的**地址** | 通过地址操作原变量 | **有效**       |

例子还演示了函数返回值是指针的情况，`return px;`语句相当于定义了一个临时变量并用`px`初始化，然后临时变量`tmp`的值成为表达式`swap(&i, &j)`的值，然后在`main`函数中又把这个值赋给了p：

```c
int *tmp = px;
```

### 23.3 指针与数组

#### 1. 酒店里的秘密

想象有一个叫 `a` 的**快捷酒店**，里面有 10 个房间（数组元素 `a[0]` 到 `a[9]`）。

##### (1) 数组名 `a` 是什么？

在 C 语言的世界里，数组名 `a` 就像是这个酒店的**固定地址**（比如：长安街 1 号）。

- **它是常量：** 酒店地址是不能移动的。所以你不能写 `a = a + 1`，你不能把长安街 1 号挪到 2 号去。这就是为什么 **`a++` 是非法的**。

##### (2) 指针 `pa` 是什么？

指针 `pa` 就像是一个**灵活的导游**（或者说经理）。

- **他是变量：** 导游手里拿着一张写着地址的纸。现在纸上写着 `a[0]` 的地址。
- **他能移动：** 当你执行 `pa++` 时，导游看了一眼纸，发现当前在 0 号房，于是他抹掉旧地址，写上了**下一个房间**的地址。

##### (3) 为什么 `pa++` 是加 4 而不是加 1？

这就更有意思了。导游（指针）非常聪明，他知道这个酒店是“整型（`int`）酒店”。

- 在 `int` 酒店里，每个房间都很大，占了 **4 个平方（4 字节）**。
- 所以当导游说“去下一间房”时，他会自动在物理地址上跳过 4 个字节。**“加 1”指的是“加一个房间的宽度”**。

------

#### 2. 深度揭秘：`a[i]` 的真面目

你可能觉得 `a[2]` 这种写法理所当然。但预处理器和编译器其实很“懒”，它们在底层只认一种操作：**指针算术**。

- 当你写 `a[2]` 时，计算机内部其实是这么算的：`*(a + 2)`。
- 意思是：从酒店门口（`a`）出发，往后走两个房间（`+2`），然后进屋拿东西（`*`）。

------

#### 3. 指针的“社交礼仪”（运算规则）

两个指针见面能做什么？我们要遵循 **"Rule of Least Surprise"（最小惊讶原则）**：

- **比大小 (`pa > a`)：** 合法。就像问：“导游，你现在是不是在酒店大门后面？”
- **相减 (`pa - a`)：** 合法。就像问：“导游，你现在的位置距离大门隔了**多少个房间**？”（结果是元素个数，不是字节数）。
- **相加 (`pa + a`)：** **非法！** 两个地址相加没有任何物理意义。就像你把“北京路 100 号”和“上海路 200 号”加在一起，除了得到一个莫名其妙的数字，没有任何用处。



### 23.4 指针与const限定符

`const`限定符和指针结合起来常见的情况有以下几种。

1. 

```
const int *a;
int const *a;
```

这两种写法是一样的，`a`是一个指向`const int`型的指针，`a`所指向的内存单元不可改写，所以`(*a)++`是不允许的，但`a`可以改写，所以`a++`是允许的。

2. 

```
int * const a;
```

`a`是一个指向`int`型的`const`指针，`*a`是可以改写的，但`a`不允许改写。

3. 

```
int const * const a;
```

`a`是一个指向`const int`型的`const`指针，因此`*a`和`a`都不允许改写。

---

指向非`const`变量的指针或者非`const`变量的地址可以传给指向`const`变量的指针

但是，指向`const`变量的指针或者`const`变量的地址不可以传给指向非`const`变量的指针

### 23.5 指针与结构体

首先定义一个结构体类型，然后定义这种类型的指针：

```c
struct unit {
	char c;
	int num;
};
struct unit *p = &u;
```

要通过指针`p`访问结构体成员可以写成`(*p).c`和`(*p).num`，为了书写方便，C语言提供了`->`运算符，也可以写成`p->c`和`p->num`。

### 23.6 指向指针的指针与指针数组

指针可以指向另外一个指针变量，称为指向指针的指针

```c
int i;
int *pi = &i;
int **ppi = &pi;
```

表达式`*ppi`取`pi`的值，表达式`**ppi`取`i`的值

---

若一个数组中每个元素都是指针类型，则称为指针数组

```c
int *a[10];
```

### 23.7 指向数组的指针与多维数组

1. **多维数组是“数组的数组”**：二维数组的每一行其实都是一个独立的一维数组。
2. **指针级别的差异**：
   - `&a`：代表整栋楼的地址。
   - `a`：代表第一层楼的地址（数组指针）。
   - `*a` 或 `a[0]`：代表第一层楼第 0 间房的地址（普通指针）。
3. **数组指针的威力**：它是为了处理“一整行”而生的。它的增量（步长）等于一整行的总字节数。

- **`int a[3][4]`**：分配了 12 个整数的连续空间。
- **`int (\*p)[4]`**：只分配了一个指针的空间，用来瞄准任何“有 4 个房间的楼层”。

### 23.8 函数类型和函数指针类型

指向函数的指针，函数指针存放的就是函数的入口地址

| 概念         | 遥控器比喻                           | C 语言实现                                        |
| ------------ | ------------------------------------ | ------------------------------------------------- |
| **函数**     | 具体的电器（电视、空调）             | `void say_hello(...) { ... }`                     |
| **函数指针** | 万能遥控器本身                       | `void (*f)(...);`                                 |
| **赋值**     | 让遥控器学习电器的信号               | `f = say_hello;`                                  |
| **调用**     | 使用遥控器操作电器                   | `f(...);`                                         |
| **核心价值** | 用一个遥控器，在不同场景控制不同电器 | **回调（Callbacks）**、**策略模式**、**事件驱动** |

**函数本身 (Function)：** 就像是一栋已经建好的**实体房屋**，比如 `int my_func(void) { ... }`。

**函数类型 (Function Type)：** 它不是房屋，而是**设计蓝图**。它规定了房子的规格，比如“必须是三室一厅、带一个院子”。

**函数指针 (Function Pointer)：** 它也不是房屋，而是一个**小本本**，上面可以记录某栋**房屋的地址**。

✅ `F f, g;` -> 相当于 `int f(void); int g(void);`

- **解读：** “项目经理，请按照 `F` 号蓝图，给我**盖两栋房子**，分别命名为 `f` 和 `g`。”
- **结果：** 编译器明白了，这是在**声明（Declare）**两个函数，`f` 和 `g`，它们都遵循 `F` 蓝图的规格。

❌ `F h(void);` -> 语法错误

- **解读：** “请盖一栋叫 `h` 的房子，它不接收参数，但它的**返回值**是**一栋 `F` 蓝图规格的房子**。”
- **为什么错？** C 语言的规则是：**你不能在物理上返回一栋房子（一个函数）。** 函数返回时，是通过寄存器或栈返回一个**值**。一个函数本身（一整段代码）太大了，无法被“返回”。
- **类比：** 你可以从银行取回“钱”（返回值），但你不能让银行把“另一个银行网点”（返回函数）打包给你带走。

✅ `F *e(void);` -> 正确

- **解读：** “请盖一栋叫 `e` 的房子，它不接收参数，但它的返回值是 `F *` 类型。”
- **`F \*` 是什么？** `F` 是蓝图，`*` 代表“地址”。所以 `F *` 就是**“符合 F 蓝图的房子的地址”**。
- **为什么对？** 这次你不是要银行打包一个网点给你，而是向银行要“另一个网点的**地址**”。这完全合法！地址就是一个简单的数值，可以被轻松返回。

### 23.9 不完全类型和复杂声明

C语言的类型分为函数类型、对象类型和不完全类型三大类

1. 对象类型分为标量类型和非标量类型。指针类型属于标量类型

2. 不完全类型是暂时没有完全定义好的类型，编译器不知道这种类型该占几个字节的存储空间，例如：

   ```c
   struct s;
   union u;
   char str[];
   ```

​	具有不完全类型的变量可以通过多次声明组成一个完全类型

```c
char str[];
char str[10];
```

如果把指针和数组、函数、结构体层层组合起来可以构成非常复杂的类型,称为复杂声明

在分析复杂声明时，要借助`typedef`把复杂声明分解成几种基本形式：

- `T *p;`，`p`是指向`T`类型的指针。
- `T a[];`，`a`是由`T`类型的元素组成的数组，但有一个例外，如果`a`是函数的形参，则相当于`T *a;`
- `T1 f(T2, T3...);`，`f`是一个函数，参数类型是`T2`、`T3`等等，返回值类型是`T1`。
